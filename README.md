# Privy Notes

Privy Notes is a secure REST API for private note management built with Spring Boot, Spring Security, JWT authentication, role-based authorization, PostgreSQL, and OpenAPI documentation.

The project demonstrates how to build a production-style backend where users can register, authenticate, manage their own profile, and create private notes. Admin users can manage users and inspect notes across the system, while regular users can only access their own data.

## Project Highlights

- Secure user authentication using Spring Security
- JWT access token and refresh token flow
- Stateless API security with OAuth2 Resource Server JWT validation
- Role-based access control with `USER` and `ADMIN` roles
- Method-level authorization using `@PreAuthorize`
- Password hashing using BCrypt
- PostgreSQL persistence with Spring Data JPA
- User-owned notes with ownership checks
- DTO-based request and response models
- Input validation using Jakarta Validation annotations
- OpenAPI and Swagger UI documentation
- Dockerized multi-stage build
- Environment-based configuration for deployment platforms such as Render

## Main Features

### Authentication and Authorization

- Users authenticate through `/authenticate` using HTTP Basic authentication.
- Successful authentication returns:
  - Access token valid for 30 minutes
  - Refresh token valid for 7 days
- `/refresh` generates a new access token from a valid refresh token.
- JWTs are signed with an RSA key pair generated by the application.
- JWT `scope` claims are converted into Spring Security role authorities.
- Invalid or expired tokens return JSON error responses.
- The application uses stateless session management.

### User Management

- Public user registration
- Duplicate username protection
- Password hashing before storage
- Current user profile endpoint
- Current user password update
- Current user account deletion
- Admin-only user listing
- Admin-only user lookup by username
- Admin-only user deletion
- User responses hide passwords and expose only safe profile data

### Notes Management

- Authenticated users can create notes.
- Notes are automatically linked to the currently authenticated user.
- Users can list only their own notes through `/notes/me`.
- Users can view, update, and delete their own notes.
- Admin users can view all notes.
- Admin users can view notes by user ID.
- Ownership checks prevent users from accessing another user's notes.
- Notes include title, content, and owner username in API responses.

### Security Model

The project uses two Spring Security filter chains:

| Filter Chain | Purpose |
| --- | --- |
| `/authenticate` | Uses HTTP Basic authentication to verify credentials and issue JWTs |
| All other routes | Uses Bearer JWT authentication and role-based authorization |

Authorization rules include:

- `POST /users/register` is public.
- `POST /authenticate` requires Basic Auth.
- `POST /refresh` is public but requires a valid refresh token in the request body.
- `GET /users` is restricted to `ADMIN`.
- `/notes/**` requires either `USER` or `ADMIN`.
- Swagger and OpenAPI endpoints are public.

## API Endpoints

### Authentication

| Method | Endpoint | Access | Description |
| --- | --- | --- | --- |
| `POST` | `/authenticate` | Basic Auth | Login and receive access and refresh tokens |
| `POST` | `/refresh` | Public with refresh token | Generate a new access token |

### Users

| Method | Endpoint | Access | Description |
| --- | --- | --- | --- |
| `POST` | `/users/register` | Public | Register a new user |
| `GET` | `/users` | Admin | Get all users |
| `GET` | `/users/me` | User/Admin | Get current user's profile |
| `GET` | `/users/{username}` | Admin | Get a user by username |
| `PATCH` | `/users/me/password` | User/Admin | Update current user's password |
| `DELETE` | `/users/me` | User/Admin | Delete current user's account |
| `DELETE` | `/users/{username}` | Admin | Delete a user by username |

### Notes

| Method | Endpoint | Access | Description |
| --- | --- | --- | --- |
| `GET` | `/notes` | Admin | Get all notes |
| `GET` | `/notes/me` | User/Admin | Get current user's notes |
| `GET` | `/notes/{id}` | Owner/Admin | Get one note |
| `GET` | `/notes/user/{userId}` | Admin | Get notes by user ID |
| `POST` | `/notes` | User/Admin | Create a note |
| `PATCH` | `/notes/{id}` | Owner/Admin | Update a note |
| `DELETE` | `/notes/{id}` | Owner/Admin | Delete a note |

## Example Requests

### Register a User

```bash
curl -X POST http://localhost:8080/users/register \
  -H "Content-Type: application/json" \
  -d '{
    "username": "sahil",
    "password": "password123",
    "role": "USER"
  }'
```

### Authenticate

```bash
curl -X POST http://localhost:8080/authenticate \
  -u sahil:password123
```

### Create a Note

```bash
curl -X POST http://localhost:8080/notes \
  -H "Authorization: Bearer YOUR_ACCESS_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "title": "Interview Prep",
    "content": "Review Spring Security and JWT concepts."
  }'
```

### Get My Notes

```bash
curl -X GET http://localhost:8080/notes/me \
  -H "Authorization: Bearer YOUR_ACCESS_TOKEN"
```

## Getting Started

### Prerequisites

- Java 25
- Maven or the included Maven wrapper
- PostgreSQL
- Docker, optional

### Environment Variables

Create a `.env` file or configure the following environment variable in your runtime environment:

```env
DB_URL=jdbc:postgresql://localhost:5432/your_database?user=your_user&password=your_password
PORT=8080
```

`PORT` is optional. If not provided, the application runs on port `8080`.

### Run Locally

```bash
./mvnw spring-boot:run
```

On Windows PowerShell:

```powershell
.\mvnw.cmd spring-boot:run
```

### Build the Project

```bash
./mvnw clean package
```

On Windows PowerShell:

```powershell
.\mvnw.cmd clean package
```

### Run Tests

```bash

```

On Windows PowerShell:

```powershell
.\mvnw.cmd test
```

## Docker

The project includes a multi-stage Dockerfile:

- Build stage uses Maven with Eclipse Temurin 25.
- Runtime stage uses Eclipse Temurin 25 JDK.
- The generated JAR is copied into the final image.
- The container exposes port `8080`.

Build the image:

```bash
docker build -t privy-notes .
```

Run the container:

```bash
docker run -p 8080:8080 \
  -e DB_URL="jdbc:postgresql://host.docker.internal:5432/your_database?user=your_user&password=your_password" \
  privy-notes
```

## Deployment Readiness

## Notes

- `SecurityConfig` and `OauthSecurityConfig` are present as alternative/experimental security configurations, but the active configuration is `JwtSecurityConfig`.
- The RSA key pair is generated at application startup, so previously issued JWTs become invalid after restarting the application. For production, keys should be stored securely and reused across restarts.
- Admin creation should be controlled carefully in a production system. The current registration DTO accepts a role, which is useful for testing and demonstration.
